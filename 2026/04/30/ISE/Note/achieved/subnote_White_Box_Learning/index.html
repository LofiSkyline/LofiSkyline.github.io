White Box Learning (白盒学习)

上述的单环境（如 DeepPerf, DaL）和跨环境学习（如 BEETLE）方法，都将可配置软件系统视为黑盒 (Blackbox)。我们只能观察配置输入和性能输出，对于内部机制一无所知。

深度学习虽然强大，但缺乏可解释性 (Interpretable)。为了获得更好的准确性、互操作性和更少的测量成本，我们可以深入研究代码 (dig into the code)，采用白盒学习。

污点分析 (Taint Analysis)

污点分析是一种传统的静态代码分析技术，通常用于安全领域（信息流分析 Information flow analysis）。

  • 目的: 跟踪系统中的敏感信息（Sources，污点源）如何流经程序，以及是否泄漏到了公共观察者（Sinks，污点汇聚点）手中。
  • 在配置性能学习中，我们不追踪安全漏洞，而是跟踪配置选项的值 (sources) 如何传播，以及它们是否到达并影响了代码中的控制流决策 (control-flow decisions, 作为sinks)。

Comprex 方法：代码引导的局部模型

Comprex 通过静态代码分析来指导局部性能模型的构建，从而降低抽样成本并提供高度的解释性。

1. 决定代码区域与分区 (Deciding code region and partitions)

  • 跟踪配置选项在每个区域 (region) 中流向控制流决策的情况。
  • 如果某个配置选项影响了区域内的控制流（例如进入 if 还是 else 分支），那么选择或取消选择该选项将导致不同的执行路径 (different execution paths)，从而影响性能。
  • 根据这些相关选项的值对区域进行分区 (Partition)。例如，如果影响该区域的二元选项有 A 和 B，理论上最多有4个分区。但通过污点分析，我们可能会发现选项 B 只在选项 A 被选中（进入特定的 if 块）时才会被执行。因此，某些分区在逻辑上是不可能的，从而大幅减少了需要采样的数量。

2. 基于分区增量采样 (Sampling with respect to partitions)

  • 利用分析得到的有效控制流依赖（重叠的子空间 subspaces），增量式地 (incrementally) 提取能够覆盖所有实际执行路径的最少配置样本集合。
  • 确保至少有一个配置样本覆盖每个必要的子空间。相比于黑盒方法，这种白盒采样策略避免了对无效或冗余组合的昂贵测量。

3. 构建局部线性模型 (Build linear model per region)

  • 为每个特定的执行区域分别建立一个线性回归模型 (linear regression)。
  • 线性模型的优势在于它高度可解释 (Interpretable)。根据在各个区域观察到的配置和测量的性能，提取出局部模型系数。
  • 全局组合: 最终系统的整体性能模型，就是将这些基于实际代码控制流推导出的局部线性模型相加（例如 $m_{global} = m_{main} + m_{foo} + m_{bar}$）。

白盒学习的优势:

  • 利用代码特征，精准指导采样，减少测量成本。
  • 构建的线性模型具有强解释性，不仅能预测性能，还能让开发者追溯到性能瓶颈在代码中的具体位置及其原因。
