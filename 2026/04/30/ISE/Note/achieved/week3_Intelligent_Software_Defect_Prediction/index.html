Intelligent Software Defect Prediction (智能软件缺陷预测)

缺陷 (Defect) 是计算机软件在设计、开发或运行过程中的错误、缺陷或故障，导致其产生不正确或意想不到的结果，或以意想不到的方式运行。缺陷可能源于对用户需求的误解、架构设计错误、代码编写错误、硬件兼容性等。

为什么要预测软件缺陷？(Why Predicting Software Defect?)

软件缺陷的修复成本极其高昂 (expensive in cost)。

  • 在编码阶段 (coding step)，捕获和纠正缺陷的成本约为 $977/defect。
  • 在软件测试阶段 (testing phase)，成本增加到 $7,136/defect。
  • 在维护阶段 (maintenance phase，即软件已上线运行)，成本飙升至 $14,102/defect（因为会涉及SLA违约惩罚、用户流失等严重后果）。
    因此，预测软件缺陷可以大幅降低开销，即省钱 (saving money)。

预测软件缺陷的挑战 (Why is it Challenging?)

  • 难以找到可靠的预测因子/特征 (reliable predictors/features)。
  • 过拟合 (Overfitting)：在某个项目上训练的模型在应用于另一个项目时质量急剧下降。
  • 不同项目有不同的需求。
  • 如何在“即时预测 (just-in-time prediction)”中进行验证。
  • 预测所需的数据量和训练成本很高。

核心知识点模块

为了预测一个类/方法/组件是否存在缺陷（构建函数 $C = f(c_1, c_2, \dots, c_n)$），本周涵盖了以下几个层面的技术与方法：

  1. subnote_Classic_Metrics_for_Defect_Prediction：探讨80年代以来的经典代码复杂度度量标准（如 McCabe 圈复杂度、Halstead 复杂度等）。
  2. subnote_Within_Project_Prediction：在项目内基于历史提交记录（Commit level）进行即时 (Just-In-Time) 缺陷预测，以及 SZZ 算法提取标签。
  3. subnote_Cross_Project_Prediction：如何解决跨项目预测中的指标不匹配 (metric mismatching) 问题，介绍了异构缺陷预测 (HDP) 和指标匹配算法。
  4. subnote_Learned_Representation_Prediction_and_XAI：使用深度信念网络 (DBN) 学习代码语义表示，以及使用 PyExplainer 实现可解释的缺陷预测 (Explainable Defect Prediction)。
