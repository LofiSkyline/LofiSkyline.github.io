Intelligent Configuration Performance Learning (智能配置性能学习)

智能软件工程 (Intelligent Software Engineering, ISE) 中的另一个关键领域是数据驱动的软件工程 (Data-Driven Software Engineering)。除了上周讨论的缺陷报告分析外，软件的运行时性能 (Runtime Performance) 也是至关重要的一环。

什么是软件配置？(What are Software Configurations?)

软件配置允许用户根据需要灵活地定制和使用软件系统。配置不仅仅是UI界面上的复选框，还包括命令行参数、环境变量、甚至深度学习模型中的超参数 (Hyperparameters)。

  • 为了满足性能需求，软件工程师必须在部署前理解在给定配置下能获得什么样的性能。
  • 这不仅有助于做出更好的配置决策，还能减少配置测试 (configuration testing) 的工作量。

为什么要学习配置性能？(Why Learning Configuration Performance?)

  1. 配置空间庞大 (High number of possible configurations)：例如，图像处理编译器 HIPAcc 拥有超过 10,000 种可能的配置。SQLite 作为一个轻量级数据库，也提供约88个编译时配置选项。
  2. 测量成本极高 (Expensive profiling)：即使配置数量较少，对单一配置进行基准测试和性能分析也可能非常昂贵（可能需要数周的运行时间）。

面临的挑战 (Why is it Challenging?)

建立配置性能模型并不容易，主要面临以下挑战：

  1. 数据集相对较小 (Relatively small dataset)：由于性能测量极其昂贵，我们能获得的数据样本量非常有限。
  2. 稀疏性 (Sparsity)：
    • 特征稀疏性 (Feature sparsity)：系统的配置性能全景图 (configuration landscapes) 通常不平滑 (not smooth)。例如，在两种缓存策略之间切换（仅改变一个变量的值）可能会极大地影响性能，导致性能曲面上出现急剧的突变。
    • 样本稀疏性 (Sample sparsity)：配置空间中的样本倾向于形成具有不同性能和配置选项值的“分区 (divisions)”。

核心知识点模块

为了应对上述挑战，本周内容分为四个主要方向，旨在构建一个准确的函数 $\mathcal{P} = f(c_1, c_2, \dots, c_n)$ 来预测给定配置的性能：

  1. subnote_Configuration_Sampling：探讨如何选择具有代表性的配置样本进行测量（二元与非二元采样策略）。
  2. subnote_Single_Environment_Learning：在单一环境下，如何处理特征编码并使用深度学习模型（如 DeepPerf, DaL）处理稀疏性问题。
  3. subnote_Cross_Environment_Learning：如何利用迁移学习 (Transfer Learning) 应对环境的剧烈变化（如 BEETLE 方法）。
  4. subnote_White_Box_Learning：打破黑盒假设，利用静态代码分析（污点分析 Taint Analysis）指导局部模型的构建（如 Comprex 方法）。
