DDS-05: Leader Election

DDS-05: Leader Election

这一讲不仅是关于如何选出 Leader，更重要的是建立了一套形式化研究分布式算法的范式。

1. Synchronous Network Model (同步网络模型)

这是研究分布式算法的“舞台”。在这个模型中，消息延迟 (message delays) 和 执行速度 (execution speed) 都是 known and bounded (已知且有界的)。

1.1 系统的图表示法 (Graph Representation)

网络被抽象为一个有向图 $G=(V, E)$，具有固定的消息字母表 $M$。

  • Nodes ($V$): 代表 processes (进程)。每个进程有唯一的 UID。
  • Edges ($E$): 代表 communication channels (通信通道)。通道 $(i,j)$ 是有向的，且在任一轮次中最多容纳一条来自 $M$ 的消息。

1.2 进程的内部结构 (Process Components)

对于进程 $i \in V$，形式化定义为五个部分：

  • $states_i$: Set of states (状态集合)（不一定有限）。
  • $start_i$: Start states (初始状态)（$states_i$ 的非空子集）。
  • $msgs_i$: Message generation function (消息生成函数)。根据当前状态决定发送给邻居的消息。
  • $trans_i$: State transition function (状态转换函数)。根据当前状态和收到的消息决定新状态。

1.3 系统的执行方式 (Execution)

系统以 lock-step (锁步/同步步调) 模式运行，每一轮 (Round) 重复以下动作：

  1. 发送阶段：各进程应用 $msgs_i$ 生成消息并放入通道。
  2. 转换阶段：各进程应用 $trans_i$ 根据收到的消息计算新状态，并清空通道。

注意：模型不计入本地计算的开销，仅关注通信轮次。

1.4 故障模型 (Fault Model)

  • Link failure: 通道丢失消息。
  • Stop failure: 进程在执行中途（如发送和转换之间）停止。
  • Byzantine failure: 进程产生任意、不可预测的消息和状态，不再遵循函数规则。

2. Leader Election Specification (问题规格)

分布式问题的正确性由两类性质定义：

  • Safety (安全性): “Nothing bad happens” —— 任意时刻系统中最多只有一个 leader。
  • Liveness (活性): “Something good eventually happens” —— 最终（或在时间界限内）系统中一定会出现一个 leader。

3. LCR Algorithm (LeLann, Chang and Roberts)

适用于单向环 (Unidirectional Ring)，假设 UID 是唯一的且支持比大小。

3.1 核心逻辑 (比大小)

  1. 每个进程将自己的 UID 发给顺时针邻居。
  2. 收到传入 UID 时：
    • 若 $UID_{in} > UID_{self}$: 继续转发该 UID。
    • 若 $UID_{in} < UID_{self}$: 丢弃该 UID。
    • 若 $UID_{in} = UID_{self}$: 该 UID 绕行全网未被拦截 $\rightarrow$ 自己就是 Leader！

3.2 复杂度分析与推导

  • Time Complexity: $n$ rounds。最大 UID 绕环一周。
  • Message Complexity: $O(n^2)$。
    • 推导与解释：
      考虑最坏情况（UID 降序排列，如 $n, n-1, …, 1$）。
      • 最大 ID ($n$) 会绕行全网 $n$ 步回到自己。
      • 第二大 ID ($n-1$) 会在遇到 $n$ 之前绕行 $n-1$ 步。
      • … 第 $i$ 大的 ID 会在遇到更大的 ID 前走 $n-i+1$ 步。
      • 总消息数 $M = \sum_{i=1}^{n} i = \frac{n(n+1)}{2} = O(n^2)$。
    • 直观理解：在最坏情况下，大量的小 ID 并没有被迅速拦截，而是跑了很长的路程才“撞上”更强者。通信开销随着节点数的增加呈平方级增长。

3.3 LCR Halting (收尾机制)

  • 缺陷: 原始 LCR 中，非 Leader 节点处于 unknown 状态，无法优雅停止。
  • 优化: Leader 确立后发出一个 finished 消息。
  • 影响:
    • Time: 增加到 $2n$ rounds。
    • Message: 依然是 $O(n^2)$。
    • 好处: 非 Leader 节点能明确知道自己是 follower，从而安全释放资源。

4. HS Algorithm (Hirschberg and Sinclair)

核心策略：“步步为营，以闲置换带宽”。要求 Bidirectional communication (双向通信)。

4.1 核心策略：Phases & Radius

在每个 Phase $l$，节点向左右两个方向各发送一个探测 token，步长为 $2^l$：

  1. Outbound (向外): 遇到更大的 UID 则被丢弃；若能走完 $2^l$ 步则转为 Inbound。
  2. Inbound (返回): 若两个方向的探测 token 都平安返回，节点进入 Phase $l+1$。
  3. Leader 判定: 若 outbound 期间收到了自己的 UID，说明半径已覆盖全环，自己是 Leader。

4.2 复杂度推导 (数学分析)

  • Time Complexity: $O(n)$。
    • 每一个 Phase $l$ 消耗的时间是 $2 \cdot 2^l = 2^{l+1}$（往返）。
    • 总时间 $T = \sum_{l=0}^{\lceil \log n \rceil - 1} 2^{l+1} + n$。根据等比数列求和，这依然被限制在 $O(n)$。
  • Message Complexity: $O(n \log n)$。
    • 详细推导 (Detailed Mathematical Derivation)：
      1. 阶段数量 (Number of Phases)：由于步长按 $2^l$ 指数级增长，总共有 $L \approx \log n$ 个阶段。
      2. 发起者数量推导 (Deriving $N_l$)：
        • 在 Phase $l$ 中，一个节点 $i$ 成为发起者的前提是它在 Phase $l-1$ 中胜出。
        • 胜出条件 (Survival Condition)：节点 $i$ 必须是其左右两侧距离 $2^{l-1}$ 范围内所有节点中的最大值。
        • 影响区间 (Segment of Maximum)：这意味着节点 $i$ 在一个长度为 $2^{l-1} (left) + 1 (self) + 2^{l-1} (right) = 2^l + 1$ 的连续片段中是唯一的最大值。
        • 间距限制 (Inter-initiator Distance)：如果两个发起者 $i$ 和 $j$ 之间的距离 $\le 2^{l-1}$，则其中 UID 较小的节点必在 Phase $l-1$ 被拦截。
        • 结论：在任何连续的 $2^{l-1} + 1$ 个节点中，最多只有一个节点能进入 Phase $l$。
        • 因此，$N_l \le \frac{n}{2^{l-1}+1}$。
      3. 消息数/发起者 (Messages per Initiator)：每个发起者在两个方向各发一个 token，每个 token 往返共 $2 \cdot 2^l = 2^{l+1}$ 步。总消息为 $2 \cdot 2^{l+1} = 2^{l+2}$。
      4. Phase $l$ 总消息数 $M_l = N_l \cdot 2^{l+2} \le \frac{n}{2^{l-1}} \cdot 2^{l+2} = n \cdot 2^3 = 8n$。
      5. 全过程总消息数 $M_{total} = \sum_{l=0}^{\log n} M_l \le 8n (\log n + 1) = O(n \log n)$。

4.3 理论下界

对于 Comparison-based (基于比较) 的算法，消息复杂度的下界是 $\Omega(n \log n)$。HS 算法已达到该类算法的理论最优解。若要达到 $O(n)$ 消息量，必须使用 Non-comparison algorithms（如 TimeSlice），但会牺牲巨大的时间开销。

5. Flooding Algorithm (通用网络选主)

适用于强连通网络 (Strongly Connected Network)。

5.1 核心逻辑

  1. 每个进程记录见过的最大 UID（初始为自己）。
  2. 每轮将当前已知的最大 UID 广播给所有邻居。
  3. 运行 $diam$ (直径) 轮后（若直径未知，则使用上界 $n$）：
    • 若 $MaxUID = UID_{self}$，则为 Leader；否则为 Non-leader。

5.2 复杂度与优化

  • Time: $diam$ rounds。
  • Message: $O(diam \cdot |E|)$。
  • 优化: 仅在学到更新、更大的 UID 时才转发消息。虽然不改变 $O$ 阶，但在密集网络中能显著减少实际流量。

6. Comprehensive Exam Question Analysis (20 Marks)

Question (Original Text)

Assume a connected, bidirectional ring network with N nodes.

(a) Define the leader election problem. [5 marks]

(b) Develop an algorithm that solves leader election in a synchronous bidirectional ring network. State any assumptions. [5 marks]

© Analyse the communication complexity of your algorithm. [5 marks]

(d) The communication complexity of leader election algorithms that do not permit the modification of process identifiers is bounded by $\Omega(n \log n)$. Explain whether you agree with this statement. Identify any implications for your algorithm. [5 marks]

Standard Answer (English Focus)

(a) Define the leader election problem

A formal specification for leader election consists of two key properties:

  • Safety: At any one time, there is at most one leader in the system. (任意时刻系统中最多只有一个 leader)
  • Liveness: Eventually (or within a time bound), there exists a leader in the system. (系统最终一定会选出一个 leader)
  • Note: If a node is elected leader, then all previous leaders must have crashed or the system must have converged to a single distinguished process.

(b) Develop an algorithm (HS Algorithm)

  • Assumptions:
    1. Synchronous network model (lock-step rounds).
    2. Bidirectional ring topology.
    3. Each process has a unique identifier (UID).
    4. Processes know their immediate left and right neighbors.
  • Algorithm Logic:
    • The algorithm proceeds in Phases $l = 0, 1, 2…$
    • In each phase $l$, a process sends tokens in both directions with a hop-count of $2^l$. (每个阶段向两端发送步长为 $2^l$ 的探测消息)
    • Comparison:
      • If $UID_{received} > UID_{self}$, it relays the message.
      • If $UID_{received} < UID_{self}$, it discards the message.
      • If $UID_{received} == UID_{self}$ during the outbound trip, it declares itself Leader.
    • Phase Transition: A process enters Phase $l+1$ only if both tokens successfully return to it after traveling $2^l$ hops. (只有探测消息平安返回，才进入下一阶段)

© Analyse the communication complexity

  • Conclusion: The communication complexity is $O(n \log n)$.
  • Analysis:
    • In phase $l$, the number of initiators $N_l$ is limited because any initiator must have won in phase $l-1$. Thus, $N_l \le \frac{n}{2^{l-1}+1}$. (阶段 $l$ 的发起者数量受限于前一阶段的胜出者)
    • Each initiator generates $2 \times 2 \times 2^l = 2^{l+2}$ messages (outbound and inbound in both directions).
    • The total messages in phase $l$ is $M_l = N_l \times 2^{l+2} \le \frac{n}{2^{l-1}} \times 2^{l+2} = 8n$. (每一阶段的消息总量被限制在 $8n$ 以内)
    • Since there are approximately $\log n$ phases, the total message complexity is $8n(\log n + 1) = O(n \log n)$.

(d) Lower bound and Implications

  • Stance: Agree.
  • Reasoning: This statement refers to the theoretical lower bound proved by Frederickson and Lynch. For comparison-based algorithms (which do not modify or swap UIDs), the message complexity is strictly bounded by $\Omega(n \log n)$ in a ring network. (对于基于比较的算法，$\Omega(n \log n)$ 是理论上的开销下界)
  • Implications:
    • It implies that the HS algorithm is asymptotically optimal among comparison-based algorithms. (HS 算法在此类算法中已达到渐进最优)
    • To achieve a better complexity (e.g., $O(n)$), one must use non-comparison based algorithms (like TimeSlice or VariableSpeeds) which manipulate the numerical properties of IDs, but these often incur a much higher Time Complexity. (若要突破该下界达到 $O(n)$，必须使用非比较算法，但这通常会牺牲时间复杂度)
