智能软件工程课程索引 (Intelligent Software Engineering - ISE Index)

本索引整理了智能软件工程 (Intelligent Software Engineering - ISE) 课程的核心学习笔记与课程大纲，涵盖了缺陷报告挖掘、缺陷预测、参数采样与机器学习软件开发方法。

📚 课程章节笔记目录 (Modules & Chapters)

🐛 Part 1: 数据驱动缺陷与配置分析 (Data-Driven Defect & Configuration Analysis)

  • ISE Week 1: Lifecycle of Bug Reports or ISE Note Week 1: Bug Report Mining — 缺陷报告分类与预处理 (Text Filtering, Stemming, TF/TF-IDF) 以及 Bug 报告的状态机生命周期。
  • ISE Note Week 2: Configuration Performance Learning (CPL) or Intelligent Configuration Performance Learning (智能配置性能学习) — 软件配置空间探索、性能测度及 Single/Cross-Environment 配置学习。
  • ISE Note Week 3: Intelligent Software Defect Prediction or Intelligent Software Defect Prediction (智能软件缺陷预测) — 缺陷预测指标辨析，不平衡数据集处理 (SMOTE, Under-sampling) 以及项目内与跨项目预测。

🧪 Part 2: 自动化测试与算法组件评估 (Automated Testing & Empirical Evaluation)

  • 第四周：自动软件测试与基于搜索的软件测试 (AST & SBST) — 遗传算法 (Genetic Algorithm) 在测试自动生成中的适应度函数 (Fitness Function) 设计。
  • 学习笔记 Week 5: Deep Learning in SE — 探讨代码的词向量表示、AST 树结构及 Transformer/LLM 在代码漏洞定位与修复中的机制。
  • Evaluation in Intelligent Software Engineering (智能软件工程中的评估) — 经验评估方法：参数敏感性、随机种子、统计假设检验 (Wilcoxon Signed-Rank Test) 及效应量 (Effect Size) 指标。

🤖 Part 3: 面向 AI 的系统开发方法 (Engineering for AI-Enabled Systems)

  • 第八周：AI 系统的需求工程 (Requirements Engineering for AI) or Requirements Engineering for AI-Enabled Systems — 探讨 AI 系统开发中的 Goal-Oriented Requirements 难题与 RE4AI 的经典规范。
  • 第九周：AI 系统架构设计 (Architectures for AI) — 机器学习工作流中的数据源设计、模型评估管线与架构妥协权衡 (Trade-offs)。

🔬 智能软件工程子主题研究 (Sub-Topic Deep Dives)

  • Intelligent Software Engineering Overview — 智能软件工程的总体定义与核心学术边界。
  • Bug Report Classification (缺陷报告分类) — 详尽解析利用机器学习对缺陷文本属性判别的模型选型。
  • Duplicate Bug Report Detection (重复缺陷报告检测) — 基于文本相似度（BM25, Cosine Similarity）的重复缺陷判定。
  • Bug Localization using Report (利用报告进行缺陷定位) — 基于信息检索（IR）与源码关联分析的 Bug 定位技术。
  • Bug Report Prioritization (缺陷报告优先级排序) — 缺陷报告重要性排序算法。
  • Configuration Sampling (配置抽样) — 随机抽样与基于边界划分的配置空间样本采集。
  • Single Environment Learning (单一环境学习) — 经典线性回归及多项式回归对单个系统运行环境配置性能的拟合。
  • Cross Environment Learning (跨环境学习) — 参数迁移方法：特征缩放与分布对齐。
  • Cross Project Prediction (跨项目预测) — 解决源项目和目标项目特征不一致的无监督域适应算法。
  • Classic Metrics for Defect Prediction (经典缺陷预测度量标准) — 详细推导 Precision, Recall, F-measure, G-measure 与 MCC 指标。
  • Learned Representation & Explainable Defect Prediction (学习表示与可解释预测) — LIME 等可解释性模型在漏洞缺陷预测中的应用。
  • White Box Learning (白盒学习) — 利用符号化树形结构进行可解释软件白盒测试评估。
  • Non-Functional Requirements for AI (AI的非功能性需求) — AI 系统的可解释性、公平性、健壮性等非功能指标保障。
  • Requirements Elicitation & Negotiation (需求的获取与协商) — 利益相关者多目标冲突的协商机制。
  • Untangling Goals in AI (梳理AI系统中的目标) — 将高阶业务目标转化为模型优化目标（Objectives）。
  • The World vs The Machine (世界与机器) — Jackson-Zave 模型：世界环境假设、系统规格与软件需求的三角关系。
