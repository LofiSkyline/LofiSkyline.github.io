Topic 10: Reliable Broadcast Algorithms

Topic 10: Reliable Broadcast Algorithms

1. 概述 (Overview)

广播（Broadcast）是分布式系统中的核心通信原语，允许一个进程向一组进程发送消息。在异步系统中，由于消息丢失和进程崩溃的可能性，我们需要不同的广播抽象来提供不同等级的可靠性保证。

Key Concepts:

  • Broadcast: The act of an application sending a message to all members of a group.
  • Deliver: The act of the broadcast abstraction handing a received message up to the application.
  • Asynchronous System: We assume no bounds on message delay or process speed, utilizing Reliable Links and Failure Detectors.

2. 基础可靠广播算法 (Reliable Broadcast Toolkit)

2.1 最佳努力广播 (Best-Effort Broadcast, BEB)

Definition (Best-Effort Broadcast): The simplest broadcast abstraction that guarantees delivery only if the sender does not crash. (最简单的广播抽象，仅在发送方不崩溃的情况下保证送达。)

Properties:

  1. Validity: If a correct process $p_i$ broadcasts a message $m$, then every correct process $p_j$ eventually delivers $m$.
  2. No Duplication: No message is delivered more than once.
  3. No Creation: No message is delivered unless it was broadcast.

Algorithm (BEB):
Uses Reliable Links (Perfect Links) to send messages to all processes.

  • Reliability: 依赖发送方的正确性。若发送方在发送过程中崩溃，可能只有部分接收者收到消息。
  • Complexity: $O(N)$ messages (where $N$ is the number of processes).

2.2 可靠广播 (Reliable Broadcast, RB)

Definition (Reliable Broadcast): A broadcast abstraction that ensures Agreement: if any correct process delivers a message, all correct processes deliver it, regardless of whether the sender crashed. (确保“协议性”：即使发送方崩溃，只要有一个正确进程交付了消息，所有正确进程都会交付该消息。)

Properties:

  1. BEB Properties: Validity, No Duplication, No Creation.
  2. Agreement: If a message $m$ is delivered by some correct process, then $m$ is eventually delivered by every correct process. (All-or-nothing guarantee).

Algorithm (Eager RB):
Processes relay the message to everyone else upon receiving it for the first time.

  • Complexity: $O(N^2)$ messages in the worst case (everyone relays to everyone).

2.3 均匀可靠广播 (Uniform Reliable Broadcast, URB)

Definition (Uniform Reliable Broadcast): A stronger version of RB where the agreement property applies to all processes (including those that later crash). If any process (faulty or correct) delivers $m$, then all correct processes must deliver $m$. (更强的保证：只要有进程（即使是故障进程）交付了消息，所有正确进程必须交付该消息。)

Properties:

  1. URB Agreement: If a message $m$ is delivered by any process (correct or faulty), then $m$ is eventually delivered by every correct process.

Algorithm (Majority-based URB):
A process delivers a message only after it knows that a majority (or all correct processes) have received it.

3. 广播算法对比分析 (Comparison & Analysis)

3.1 RB vs. URB (The “Omniscience” Problem)

  • RB (Reliable Broadcast): 只保证正确进程之间的一致性。如果一个进程在交付消息后立刻崩溃，而它又是唯一收到消息的进程，RB 并不要求其他正确进程也交付该消息。
  • URB (Uniform Reliable Broadcast): 保证任何进程（即使随后崩溃）交付了消息，正确进程也必须交付。这在涉及外部副作用（如打印支票、触发硬件动作）时至关重要。

3.2 复杂度对比 (Complexity Summary)

Algorithm Message Complexity (Worst Case) Latency (Message Steps)
BEB $O(N)$ 1
RB (Eager) $O(N^2)$ 2
URB $O(N^2)$ 2+ (requires acks)

4. 排序广播 (Ordered Broadcast)

排序广播在可靠广播的基础上增加了对消息交付顺序的限制。

4.1 FIFO 广播 (FIFO Broadcast)

Definition (FIFO Order): If a process $p_i$ broadcasts message $m_1$ before $m_2$, then no correct process delivers $m_2$ unless it has already delivered $m_1$. (单源顺序：同一发送方发出的消息必须按发送顺序交付。)

FIFO vs. Global FIFO:

  • FIFO: 只保证来自同一个发送者的顺序。不同发送者的消息在不同接收者那里的顺序可能不同。
  • Global FIFO (Causal): 保证全局因果顺序。

Algorithm (FIFO RB):
Uses a sequence number per sender.

4.2 因果广播 (Causal Order Broadcast, COB)

Definition (Causal Order): If the broadcast of $m_1$ causally precedes the broadcast of $m_2$ ($m_1 \to m_2$), then no correct process delivers $m_2$ unless it has already delivered $m_1$. (因果顺序：若消息 $m_1$ 导致了 $m_2$ 的产生，则 $m_1$ 必须在 $m_2$ 之前被交付。)

Algorithm (Vector Clock Based COB):
这是实现 COB 的最常用方法。每个进程维护一个向量时钟 $VC$。

  • Broadcast $m$: 增加自己的 $VC[i]$，将 $VC$ 随消息 $m$ 发出。
  • Deliver $m$ (from $p_j$ with $VC_m$): 满足以下条件时交付：
    1. $VC_m[j] = VC_{local}[j] + 1$ (这是 $p_j$ 发出的下一条消息)。
    2. 对于所有 $k \neq j$，$VC_m[k] \leq VC_{local}[k]$ (接收者已经交付了所有 $p_j$ 在发送 $m$ 之前看到的因果相关消息)。

Case Study (案例分析):

  1. $P_1$ 广播 $m_1$ ($VC=[1,0,0]$)。
  2. $P_2$ 收到 $m_1$，更新本地 $VC=[1,0,0]$。
  3. $P_2$ 广播 $m_2$ ($VC=[1,1,0]$)。
  4. $P_3$ 先收到 $m_2$。检查条件：$VC_{m2}[1]=1 > VC_{local}[1]=0$。阻塞 $m_2$ 的交付。
  5. $P_3$ 收到 $m_1$，交付 $m_1$，更新 $VC_{local}=[1,0,0]$。
  6. $P_3$ 现在可以交付 $m_2$，因为条件 2 满足。

5. 总结

特性 BEB RB URB FIFO COB
可靠性 弱 强 (正确进程一致) 极强 (所有进程一致) 强 强
顺序 无 无 无 单源顺序 全局因果顺序
实现基础 Perfect Links BEB RB + Acks RB + SN RB + Vector Clocks
