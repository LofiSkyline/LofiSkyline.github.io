DDS-04: Dependability Analysis and Evaluation (可靠性分析与评估)

DDS-04: Dependability Analysis and Evaluation (可靠性分析与评估)

1. 核心概念 (Core Concepts)

在系统开发的早期进行可靠性分析可以尽早发现权衡点，避免后期重新设计的巨大成本。

  • Hazard (危险): 系统的状态或条件集合，结合环境条件将导致事故 (an accident)。通常根据我们要避免的状态来定义。
  • Risk (风险): 是 hazard level（危险严重程度加上发生的可能性）与危险实际引发事故的可能性的结合。
  • Safety Case (安全案例): 一份清晰、综合的论证，用于证明一个系统在特定的上下文中运行是可接受的安全的 (acceptably safe to operate in a particular context)。

2. 可靠性数学公式 (Reliability Mathematical Formulas)

以下是用于计算和量化系统可靠性的核心数学公式。

2.1 指数失败理论 (The Exponential Failure Law)

当设备在其有用生命周期 (useful phase) 内，失败率 (failure rate) 被假设为一个常数 $\lambda$。此时，系统的可靠性随时间呈指数下降：
$$R(t) = e^{-\lambda t}$$

2.2 Mean Time To Failure (MTTF - 平均无故障时间)

MTTF 是系统在第一次发生故障前预计能运行的平均时间。它是可靠性对时间的积分（也就是失败密度函数的期望值）：
$$MTTF = \int_{0}^{\infty} R(t) dt \Rightarrow MTTF = \int_{0}^{\infty} e^{-\lambda t} dt = \frac{1}{\lambda}$$

2.3 Mean Time To Repair (MTTR - 平均修复时间)

MTTR 是修复发生故障的计算机系统所需的平均时间（包含检测、定位、修复和重配置）。类似失败率，修复率 (repair rate) 记为 $\mu$：
$$MTTR = \frac{1}{\mu}$$

2.4 Mean Time Between Failures (MTBF - 平均故障间隔时间)

MTBF 是两次故障之间流逝的平均时间。假设系统在修复后恢复到原状态（无替代），则：
$$MTBF = MTTR + MTTF$$

3. 组合模型 (Combinatorial Modelling)

任何实际系统最终都可以简化为一系列串联和并联的组合，通过可靠性区块图 (Reliability Block Diagram, RBD) 来抽象表示。

3.1 Series Systems (串联系统)

所有组件都必须正常工作，系统才能正常运行。
$$P(S_t) = R_1(t) \times R_2(t) \times … \times R_N(t) = \prod_{i=1}^{N} R_i(t)$$

3.2 Parallel Systems (并联系统)

只要有至少一个组件正常工作，系统就不会失败。只有全部失败系统才会失败。
$$P(S_t) = 1 - \prod_{i=1}^{N} (1 - R_i(t))$$

3.3 M-of-N Systems (M/N 系统)

This is a generalisation of parallel systems. Instead of at least one component working, an M-of-N system requires that at least $M$ out of $N$ components must work for the overall system to operate successfully.
这可以看作是并联系统的泛化形式。在总共 $N$ 个组件中，至少需要 $M$ 个组件正常工作，系统才能维持运行。

The reliability of such a system is expressed using the binomial probability formula (基于二项分布的概率公式):
$$P(S_t) = \sum_{i=0}^{N-M} \frac{N!}{(N-i)!i!} (R(t))^{N-i} (1 - R(t))^i$$

参数与推导步骤详解 (Parameter Breakdown & Logical Steps):

  • $N$: Total number of components in the system (系统中的总组件数).
  • $M$: Minimum number of components required to be operational (系统正常运行所需的最小正常组件数).
  • $i$: The number of failed components (发生故障的组件数量). The summation allows $i$ to range from $0$ (no failures) up to $N-M$ (the maximum tolerable failures). 累加符号表示允许系统存在 0 到 $N-M$ 个坏掉的组件。
  • $\frac{N!}{(N-i)!i!}$: This is the binomial coefficient $C_N^i$ (or $N \text{ choose } i$), representing the number of ways to choose $i$ failing components out of $N$. (从 $N$ 个组件中挑选 $i$ 个坏掉组件的组合数).
  • $(R(t))^{N-i}$: The probability that the remaining $N-i$ components are functioning correctly. ($N-i$ 个组件保持正常的概率).
  • $(1 - R(t))^i$: The probability that the specific $i$ components have failed. ($i$ 个组件发生故障的概率).

3.4 三模冗余 (Triple Mode Redundancy, TMR) 案例

Three processors run in parallel, and their outputs are compared by a voter. Voting is done according to the majority, thus at least two out of three must work correctly.
三个处理器并行运行，输出通过一个表决器进行比较（少数服从多数）。这是一个典型的 2-of-3 系统 ($N=3, M=2$).

假设每个处理器的失败率为 $\lambda$，根据指数失败理论，单个处理器的可靠性 $R(t) = e^{-\lambda t}$。代入 M-of-N 公式，由于最多允许坏掉 $3-2=1$ 个处理器，所以 $i$ 取 $0$ 和 $1$：
$$P(S_t) = \underbrace{\frac{3!}{3!0!} ((e^{-\lambda t})^3)(1 - e^{-\lambda t})^0}{\text{All 3 processors working } (i=0)} + \underbrace{\frac{3!}{2!1!} ((e^{-\lambda t})^2)(1 - e^{-\lambda t})^1}{\text{2 working, 1 failed } (i=1)}$$
其 $MTTF$ 计算结果为 (通过对 $P(S_t)$ 积分 $\int_0^{\infty} P(S_t) dt$)：
$$MTTF = \frac{5}{6\lambda}$$

4. 最小割集 (Minimal Cut Sets)

  • Cut Set (割集): 在 RBD 中，一组组件的组合，如果这些组件同时发生故障 (simultaneous failures)，就会导致整个系统发生故障。
  • Minimal Cut Set (最小割集): 是一个特定的割集，它的任何严格子集 (strict subset) 都不会导致系统故障。它为系统的可靠性提供了下界 (Lower bound on reliability)。

不同系统的最小割集特性：

  • 串联系统: 每个最小割集的大小为 1 (任何单一组件故障都会拖垮系统)。
  • 并联系统: 每个最小割集的大小等于并联组件的总数 (所有组件必须同时坏掉)。

TMR (三模冗余) 结合案例:
在包含处理器 $P_1, P_2, P_3$ 的 TMR 系统中：

  • 因为至少需要 2 个正常，所以如果有 2 个坏掉系统就崩溃了。
  • 组合 ${P_1, P_2}$, ${P_1, P_3}$, ${P_2, P_3}$ 都是导致崩溃的组合，且其子集单一组件坏掉系统不会崩溃，所以它们是最小割集。
  • 组合 ${P_1, P_2, P_3}$ (全坏) 会导致崩溃，它是割集，但不是最小割集（因为它的子集 ${P_1, P_2}$ 已经能让系统崩溃了）。

5. 马尔科夫模型 (Markov Models) 与 CTMC

马尔科夫过程基于“无记忆性 (memoryless)”，最常用于可靠性分析的是连续时间马尔科夫链 (Continuous Time Markov Chains, CTMC)，其状态空间离散，时间连续。
通常我们定义**状态 (State)**为故障节点的数量，**状态转移 (State transition)**由节点的故障率 ($\lambda$) 和修复率 ($\mu$) 决定。

2.5 概率近似 (Probability Approximation)

在构建动态模型（如马尔科夫链）时，我们经常关注极短时间间隔 $\Delta t$ 内的状态变化。
根据指数失效定律，在时间 $t$ 正常的前提下，到 $t + \Delta t$ 发生失效的概率为：
$$P(\text{failure in } \Delta t) = 1 - \frac{R(t+\Delta t)}{R(t)} = 1 - e^{-\lambda \Delta t}$$

泰勒展开近似 (Taylor Expansion):
当 $\lambda \Delta t$ 极小时，根据泰勒级数 $e^x = 1 + x + \frac{x^2}{2!} + …$，我们可以取一级近似：
$$1 - e^{-\lambda \Delta t} \approx 1 - (1 - \lambda \Delta t) = \lambda \Delta t$$
考试建议： 在构建转移矩阵或推导状态转移概率时，直接使用 $\lambda \Delta t$ 可以极大地简化计算过程。

3. 组合模型 (Combinatorial Modelling)

5.1 TMR 系统的 CTMC 建模

在构建浓缩马尔科夫模型时，我们利用上述近似来定义状态间的转移概率。这种方法比使用指数函数 $e^{-\lambda t}$ 更直观，特别是在描述状态演变时。

核心假设：单一事件原则

在极短的时间间隔 $\Delta t$ 内，我们假设最多只能发生一个事件（故障）。

  • 数学依据： 同时坏掉两个节点的概率是 $(\lambda \Delta t)^2$，这是高阶无穷小，在建模时忽略不计。
  • 结论： 系统只能在相邻状态间转移（3 $\rightarrow$ 2 $\rightarrow$ F），不能从 3 直接跳到 F。

状态转移与保持概率

  • 从状态 3 开始 (3 nodes up):
    • 转移 (To State 2): 3 个节点中任意一个坏掉的概率为 $3\lambda \Delta t$。
    • 保持 (Stay in 3): 没有任何节点坏掉的概率为 $1 - 3\lambda \Delta t$。
  • 从状态 2 开始 (2 nodes up):
    • 转移 (To State F): 剩余 2 个节点中任意一个坏掉导致系统失效，概率为 $2\lambda \Delta t$。
    • 保持 (Stay in 2): 剩余节点均正常的概率为 $1 - 2\lambda \Delta t$。

状态转移矩阵 (Transition Matrix)

利用 $\lambda \Delta t$ 线性近似，我们可以直接写出系统在 $t+\Delta t$ 时刻的状态分布：
$$
\begin{pmatrix}
P_3(t+\Delta t) \
P_2(t+\Delta t) \
P_F(t+\Delta t)
\end{pmatrix} =
\begin{pmatrix}
1-3\lambda\Delta t & 0 & 0 \
3\lambda\Delta t & 1-2\lambda\Delta t & 0 \
0 & 2\lambda\Delta t & 1
\end{pmatrix}
\begin{pmatrix}
P_3(t) \
P_2(t) \
P_F(t)
\end{pmatrix}
$$

系统的可靠性计算 (System Reliability)

在 TMR 系统中，只要有 2 个或 3 个节点正常，系统就处于“工作状态”。
因此，系统在时刻 $t$ 的可靠性 $R(t)$ 等于处于所有正常状态概率的总和：
$$R(t) = P_3(t) + P_2(t)$$
或者利用补集思想（总概率 1 减去失败状态的概率）：
$$R(t) = 1 - P_F(t)$$

5.2 流量平衡与微分方程 (Flow Balance & Differential Equations)

“流入 - 流出”原则是理解马尔科夫链动态演变的直觉核心。我们可以将概率看作在不同状态（水桶）之间流动的“流体”。

核心直觉：概率流

任何一个状态概率的变化率 $\frac{dP_i(t)}{dt}$ 遵循以下公式：
$$\text{概率变化率} = \sum (\text{流入该状态的速率}) - \sum (\text{流出该状态的速率})$$

从 $\Delta t$ 近似到微分方程的推导 (以状态 2 为例)

  1. 写出离散方程：
    $$P_2(t+\Delta t) = \underbrace{P_2(t)}{\text{原本概率}} + \underbrace{3\lambda \Delta t P_3(t)}{\text{从 3 流入}} - \underbrace{2\lambda \Delta t P_2(t)}_{\text{流向 F}}$$
  2. 移项并构造导数：
    将 $P_2(t)$ 移到左边并除以 $\Delta t$：
    $$\frac{P_2(t+\Delta t) - P_2(t)}{\Delta t} = 3\lambda P_3(t) - 2\lambda P_2(t)$$
  3. 取极限 $\Delta t \to 0$：
    得到状态 2 的微分方程：
    $$\frac{dP_2(t)}{dt} = 3\lambda P_3(t) - 2\lambda P_2(t)$$

TMR 系统微分方程组

根据此逻辑，我们可以写出描述整个系统演变的方程组：

  • 状态 3 (只有流出): $\frac{dP_3(t)}{dt} = -3\lambda P_3(t)$
  • 状态 2 (有进有出): $\frac{dP_2(t)}{dt} = 3\lambda P_3(t) - 2\lambda P_2(t)$
  • 状态 F (只有流入): $\frac{dP_F(t)}{dt} = 2\lambda P_2(t)$

概率守恒 (Conservation of Probability)

如果你将上述三个方程相加，结果恒等于 $0$：
$$\frac{d}{dt}(P_3 + P_2 + P_F) = -3\lambda P_3 + (3\lambda P_3 - 2\lambda P_2) + 2\lambda P_2 = 0$$
这证明了系统总概率始终保持为 1，没有任何概率无故消失。

5.3 从微分方程到解析解 (Analytical Solutions)

许多同学不理解为什么“流量”方程最后会变成带 $e$ 的指数式。这是微积分中“从局部变化推导全局结果”的过程。

1. 求解 $P_3(t)$ (最简单的解)

方程：$\frac{dP_3(t)}{dt} = -3\lambda P_3(t)$

  • 直觉： 变化率正比于自身。只有指数函数满足此特性。
  • 解： $P_3(t) = C \cdot e^{-3\lambda t}$
  • 初始条件： $t=0$ 时 3 个节点全好，$P_3(0)=1 \Rightarrow C=1$。
  • 结论： $P_3(t) = e^{-3\lambda t}$

2. 求解 $P_2(t)$ (积分因子法)

方程：$\frac{dP_2(t)}{dt} + 2\lambda P_2(t) = 3\lambda P_3(t)$
将 $P_3(t) = e^{-3\lambda t}$ 代入：
$$\frac{dP_2(t)}{dt} + 2\lambda P_2(t) = 3\lambda e^{-3\lambda t}$$

  • 步骤： 两边同时乘以积分因子 $e^{2\lambda t}$，左边凑成乘积导数形式：
    $$\frac{d}{dt}(P_2(t) e^{2\lambda t}) = 3\lambda e^{-3\lambda t} \cdot e^{2\lambda t} = 3\lambda e^{-\lambda t}$$
  • 积分： $P_2(t) e^{2\lambda t} = \int 3\lambda e^{-\lambda t} dt = -3 e^{-\lambda t} + C$
  • 初始条件： $t=0$ 时系统不可能只有 2 个节点，$P_2(0)=0 \Rightarrow 0 = -3 + C \Rightarrow C=3$。
  • 结论： $P_2(t) = 3e^{-2\lambda t} - 3e^{-3\lambda t}$

3. 最终 TMR 可靠性公式

$$R(t) = P_3(t) + P_2(t) = e^{-3\lambda t} + (3e^{-2\lambda t} - 3e^{-3\lambda t}) = 3e^{-2\lambda t} - 2e^{-3\lambda t}$$

(注：通过微分方程推导出的解析解，完美描述了系统随时间增长的失效过程。考试中，如果能展示从方程到解的关键步骤，将是非常有力的论证。)

(考试注：展示这种“流量平衡”的思维能证明你深刻理解了马尔科夫过程的物理意义，而不仅仅是在代入公式。)

(考试注：使用 $\lambda \Delta t$ 代替 $1 - e^{-\lambda \Delta t}$ 能清晰地展示状态间的流量平衡，且避开了复杂的微积分公式，是老师推荐的推导方式。)

转移与保持的概率 (Transition and Retention Probabilities):
根据指数失败理论展开，在极短的时间间隔 $\Delta t$ 内，单个节点故障的概率近似为 $\lambda \Delta t$。

  • 状态 3 (State 3):
    • 转移 (Transition to 2): 有 3 个节点可能发生单一故障，概率为 $3\lambda \Delta t$.
    • 保持 (Stay in 3): 系统没有任何节点坏掉，概率为 $1 - 3\lambda \Delta t$.
  • 状态 2 (State 2):
    • 转移 (Transition to F): 剩下的 2 个节点中任意一个发生故障，系统即进入失败状态，概率为 $2\lambda \Delta t$.
    • 保持 (Stay in 2): 两个节点都没有坏掉，概率为 $1 - 2\lambda \Delta t$.
  • 状态 F (State F):
    • 保持 (Stay in F): 一旦进入失败状态，系统不再恢复 (absorbing state)，保持概率为 $1$.

微分方程与最终概率求解 (Differential Equations & Final Probabilities):
利用“累积率 = 流入率 - 流出率” (Rate of Buildup = Rate of Flow In - Rate of Flow Out)，我们可以构建微分方程组：
$$ \frac{dP_3(t)}{dt} = -3\lambda P_3(t) $$
$$ \frac{dP_2(t)}{dt} = 3\lambda P_3(t) - 2\lambda P_2(t) $$
$$ \frac{dP_F(t)}{dt} = 2\lambda P_2(t) $$

假设 $t=0$ 时系统在状态 3，求解上述方程可得每个状态随时间的概率 (Transient Analysis):

  1. $P_3(t)$: 保持全好的概率
    $$P_3(t) = e^{-3\lambda t}$$
  2. $P_2(t)$: 降级但可用的概率
    $$P_2(t) = 3e^{-2\lambda t} - 3e^{-3\lambda t}$$
  3. $P_F(t)$: 系统彻底失败的概率
    $$P_F(t) = 1 - (P_3(t) + P_2(t)) = 1 - (3e^{-2\lambda t} - 2e^{-3\lambda t})$$

example

如果在考试中，题目给了你某一时刻 $t$ 的原始概率（比如 $P_3(t)=0.9, P_2(t)=0.08$），并且给了你 $\lambda$ 和一个具体的 $\Delta
t$，你不需要去解复杂的微积分方程，直接代入你写的那个离散公式就能算出下一时刻的概率。

这个方法在数学上叫 “欧拉法 (Euler’s Method)”，它是计算机模拟马尔科夫链的最基本手段。

考试中的具体算法如下：

  1. 算新的 $P_3$（只减不加）
    $$P_3(t+\Delta t) = P_3(t) - (3\lambda \Delta t) \times P_3(t)$$
    理解：老概率减去流出的部分。

  2. 算新的 $P_2$（有加有减）
    $$P_2(t+\Delta t) = P_2(t) + \underbrace{(3\lambda \Delta t) \times P_3(t)}{\text{从 3 掉下来的}} - \underbrace{(2\lambda \Delta t)
    \times P_2(t)}    {\text{掉进 F 里的}}$$
    理解：老概率加上流入的，减去流出的。

  3. 算新的 $P_F$（只加不减）
    $$P_F(t+\Delta t) = P_F(t) + (2\lambda \Delta t) \times P_2(t)$$
    理解：老概率加上新掉进来的。
    举个简单的考试例子：
    假设 $\lambda = 0.01$，$\Delta t = 1$ 秒。当前时刻 $t$，系统状态是 $P_3=1, P_2=0, P_F=0$（即刚开始运行）。

1秒后（$t+1$）的状态：

  • $P_3(1) = 1 - (3 \times 0.01 \times 1) \times 1 = \mathbf{0.97}$
  • $P_2(1) = 0 + (3 \times 0.01 \times 1) \times 1 - 0 = \mathbf{0.03}$
  • $P_F(1) = 0 + 0 = \mathbf{0}$
  • 总和：$0.97 + 0.03 + 0 = 1$（正确）

再过1秒（$t+2$）的状态：

  • $P_3(2) = 0.97 - (0.03 \times 0.97) = 0.9409$
  • $P_2(2) = 0.03 + (0.03 \times 0.97) - (2 \times 0.01 \times 1 \times 0.03) = 0.03 + 0.0291 - 0.0006 = 0.0585$
  • …以此类推。

总结：
这种“一步步迭代”的方法就是马尔科夫矩阵的本质。

  • 矩阵写法：$\mathbf{P}(t+\Delta t) = \mathbf{M} \times \mathbf{P}(t)$
  • 这里的矩阵 $\mathbf{M}$ 里的元素，就是你公式里的那些 $(1-3\lambda\Delta t)$ 等系数。
