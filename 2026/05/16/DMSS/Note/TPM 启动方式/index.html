TPM 启动与工作方式 (TPM Boot Sequence & FDE Sealing)

TPM 启动与工作方式 (TPM Boot Sequence & FDE Sealing)

Question: Explain how the TPM-based Measured Boot works and how it protects hard drive data from unauthorized access.

1. Exam Answer Template (高分英文答题模板)

Phase 1: Measured Boot (The Chain of Trust)

The TPM-based Measured Boot establishes a Chain of Trust by sequentially measuring each boot component before execution.

  1. CRTM Execution: Upon power-on, the main CPU executes the Core Root of Trust for Measurement (CRTM) embedded in its immutable BootROM. The CRTM hashes the BIOS/UEFI.
  2. The “Extend” Operation: The CRTM sends this hash to the TPM chip to “Extend” PCR[0]. Because PCRs are Extend-Only ($PCR_{new} = \text{Hash}(PCR_{old} \parallel \text{New_Data})$), attackers cannot revert or reset the logs.
  3. Chain Handoff: The verified BIOS then hashes the Bootloader (extending another PCR), and the Bootloader hashes the OS Kernel. The TPM acts purely as a passive, tamper-proof logger during this phase.

Phase 2: Sealing and Enforcement (Data Protection)

The TPM does not actively block malicious code from booting; instead, it enforces security through Sealing:

  1. Sealing: The Full Disk Encryption (FDE) Volume Master Key (VMK) is securely stored (sealed) inside the TPM’s NVRAM, bound to a set of Expected PCR values (a “known-good” system state).
  2. Unsealing (The Match): When the OS attempts to mount the encrypted drive, it requests the VMK. The TPM internally compares the Current PCR values (generated by the boot sequence) against the Expected PCR values.
  3. Enforcement: If an attacker boots from an external USB or alters the bootloader, the final PCR values will diverge. The TPM detects the mismatch and refuses to Unseal the key, leaving the hard drive permanently encrypted.

2. 核心逻辑与中文记忆辅助 (Memory Aids)

  • 物理位置的纠错：CRTM（第一发令枪）并不在 TPM 芯片内部，它烧死在主板 CPU 的 BootROM 里。TPM 本身只是个接收 Hash 并记录的“被动黑盒”。
  • TPM 的本质：TPM 在启动过程中不拦截黑客！它只会如实把黑客篡改的痕迹用 Extend 函数算进 PCR 里。真正的拦截发生在交出解密硬盘的钥匙（Unseal）时。
  • Hash 魔法与现实冲突（独立考点）：Hash 是确定性的，环境不变 PCR 就不变。但当 Windows 正常更新 Kernel 时，Hash 必然改变。此时，Windows 必须暂停 BitLocker，带着新的更新跑一遍，得出全新的一组 PCR 值，然后再将硬盘密钥贴上这组新 PCR 作为条件，重新锁回 TPM 中 (Re-sealing)。这也是更新时严禁关机的原因（否则新旧账对不上，导致硬盘彻底锁死）。
  • TPM 与内存的关系：TPM 解封后，密钥 (VMK) 被直接送到电脑主内存 (RAM) 中明文存放，由主 CPU 进行解密计算。因此它防不住液氮冷启动攻击或 DMA 内存窃取。
