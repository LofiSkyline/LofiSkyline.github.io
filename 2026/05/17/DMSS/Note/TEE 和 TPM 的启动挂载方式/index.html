TEE 和 TPM 的启动挂载方式

这份为你精心提炼的 TPM 与 TEE 启动、挂载及密钥流动 的复习笔记，完美覆盖了你们本学期大纲的核心考点。你可以直接复制保存，打印或记录下来：

📌 TPM vs TEE 启动与挂载磁盘核心笔记

1. 静态存储状态 (Static Storage State - 关机断电时)

由于 TPM / TEE 芯片内部的非易失性存储空间（NVRAM）极其有限，磁盘加密密钥不会死板地直接存在芯片内部：

  • 所有的密钥（DEK / DE Key / CE Key）在静止状态下，都是以加密密文块（Wrapped Key Blob）的形态存放在硬盘的明文元数据（Disk Metadata）区域。

  • 任何黑客如果直接物理拔走硬盘，由于拿不到硬件里的解密根密钥，看到的全部是一堆乱码。

2. TPM 挂载范式：全盘加密 (Full Disk Encryption - FDE)

  • 启动阶段 (Boot Phase): 采用 Measured Boot (度量启动)。从 CRTM (信任根) 开始，系统通电后像滚雪球一样，把每一层引导代码的哈希值不停地拼接到 TPM 的 PCR 寄存器中： New PCR = Hash(Old PCR + Current Stage Hash) 注：即使某一步被篡改，TPM 也不会拦截，只负责被动记录，允许系统继续启动。

  • 触发挂载点 (Mounting Point): 当启动走到 Initramfs (挂载根文件系统) 阶段，系统向 TPM 发起 Key Unsealing 请求。

  • 决策与解锁 (Decision): TPM 芯片在硬件内部比对当前的 PCR 数值与预设的 Predefined PCR code：

    • 若匹配： 证明底层系统纯洁未被篡改，TPM 将 DEK (Data Encryption Key) 释放给操作系统 Kernel。

    • 若不匹配： 拒绝释放密钥，全盘无法解密，系统直接卡死。

3. TEE 挂载范式：文件级加密 (File-Based Encryption - FBE)

  • 启动阶段 (Boot Phase): 采用 Verified Boot (安全验证启动)。依据 Chain of Trust (信任链)，利用硬件固化的 OEM Public Key 对每一层代码的数字签名（Digital Signature）进行严格验签。只要一步错，当场击毙（Halt）。

  • 世界隔离 (World Isolation): 启动之初，硬件逻辑直接划分好 Secure World (SW, 安全世界) 和 Normal World (NW, 普通世界)。

  • 解密与运行时隔离 (Runtime Isolation - 核心考点):

    • TEE 利用其硬件根密钥，在 SW 内部 将硬盘 Metadata 里的磁盘密钥明文 Unwrap 解开。

    • 致命红线：明文密钥永远留在 SW 内存里，绝对不流向不可信的 NW 内存。

  • 流式解密机制 (Streaming / Inline Decryption):

    • 当 NW 的普通 OS 需要读取加密文件时，硬盘吐出 Ciphertext (密文数据块)。

    • 密文流经底层的 Hardware Inline Crypto Engine (硬件内联加密引擎)。

    • 锁在 SW 内部的密钥作为实时翻译官，在硬件总线层瞬间将密文洗成 Plaintext (明文数据块) 送入 NW 内存供 CPU 执行。

4. FBE 内部的双重密钥门禁：DE Key vs CE Key

在 TEE 常驻运行后，针对不同的文件区域，解锁时机分为两波：

密钥类型 核心凭证依赖 (Key Dependency) 解锁时机与状态 (Trigger Stage) 典型应用场景 (Use Cases)
DE Key
(Device Encrypted)		 仅依赖 Hardware-backed Root Key。 BFU (Before First Unlock)
开机验证通过后，TEE 内部自发调用，无需用户参与。		 连 WiFi、启动锁屏、后台闹钟、电话拨号盘。
CE Key
(Credential Encrypted)		 依赖 Hardware Key + User Passcode。 AFU (After First Unlock)
系统停在锁屏，必须等待用户输入口令，每一次解锁时触发。		 用户个人隐私数据、微信相册、应用账号数据。
  • CE Key 的运行时生成逻辑： 用户输入 Passcode \rightarrow 送入 TEE 内部 \rightarrow 结合硬件密钥通过 KDF (Key Derivation Function) 进行几千次哈希拉伸（Key Stretching） \rightarrow 派生出 CE Key \rightarrow 激活加密引擎开启流式解密。

💡 考场高分金句（可直接背诵套用）

  1. 关于完整性 (Integrity) 的边界： > TPM 只能保证 Static Integrity (静态完整性 / 加载时完整性)，即保证代码“读进内存那一刻”没被改；它无法监控系统跑起来之后的 Runtime Integrity (运行时完整性)，也完全无法保证系统的 Availability (可用性)。

  2. 关于 TPM 与 TEE 的强强联合： > 现代可信架构不孤立使用单一技术。TPM 负责“静态的完整性与远程度量（Platform Attestation）”，确保平台干净；TEE 负责“动态的机密性与运行隔离（Memory Isolation）”，利用 SW 隔离对抗来自恶意管理员或外部黑客的运行时内存嗅探（Memory Scraping）。
