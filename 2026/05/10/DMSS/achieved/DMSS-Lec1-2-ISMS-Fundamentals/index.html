信息安全管理体系 (ISMS) 基础

信息安全管理体系 (ISMS) 基础

1. 哲学转变：从 Ad-hoc 到 Systematic

在安全管理中，最常见的失败源于 Ad-hoc Security (临时性安全)。

  • 特征：想到什么做什么，缺乏一致性。
  • 缺陷：无法证明其有效性，且在人员变动后极易崩溃。
    真正的安全管理需要通过 ISMS (信息安全管理体系) 来实现，确保安全措施是可重复、可衡量的。

“Management systems are there to replace the ‘hero culture’ where security depends on one person knowing everything.” (Transcript wk1-1)

2. 安全的三大支柱 (CIA Triad)

  • Confidentiality (机密性): 核心在于控制 Read 权限。不仅是加密，还涉及物理访问控制。
  • Integrity (完整性): 核心在于控制 Change 权限。确保数据在存储和传输中未被未授权篡改。
  • Availability (可用性): 确保系统在业务需要时 Accessible。

3. PDCA 循环：管理体系的心脏

PDCA 循环将安全从“静态设置”变为“动态过程”。

P: Plan (计划) —— 决定“做什么”

  • 资产识别: 我们要保护什么？
  • 风险与威胁评估: 谁想破坏？可能发生什么？
  • 控制措施选择: 针对性选择 控制措施 (Controls)。

D: Do (实施) —— 运行系统

  • 核心在于 程序 (Procedures) 的落实。如果没有文档化的程序，实施就是盲目的。

C: Check (检查) —— 验证有效性

这是 ISMS 与临时措施的最大区别。

  • Metrics (指标): 定量分析。如：IDS 拦截了多少次攻击？
  • Audit (审计): 定性分析。如：员工是否真的遵守了离岗锁屏的规定？
  • 目的: 衡量 Effectiveness (是否奏效) 和 Compliance (是否合规)。

A: Act (改进) —— 闭环优化

  • 根据审计结果调整计划，确保系统具备自愈和进化能力。

4. 治理与文档金字塔 (Documentation Pyramid)

为了实现可审计性，必须建立四层文档体系：

  1. Policies (策略): 高层愿景（如：移动设备使用政策）。
    • 特点: 短小、稳定、由高级管理层签发。
  2. Procedures (程序): 操作手册（如：如何重置密码）。
    • 作用: 消除歧义，确保不同员工执行标准一致。
  3. Work Instructions (工作说明书): 极度详细的技术步骤。
  4. Quality Records (质量记录): 审计证据（如：日志、审批表格）。没有记录就等于没做。
