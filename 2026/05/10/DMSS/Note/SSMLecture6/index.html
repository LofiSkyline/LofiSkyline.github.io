Lecture 6: Defence in Depth & Attack Trees (纵深防御与攻击树)

Lecture 6: Defence in Depth & Attack Trees (纵深防御与攻击树)

1. 纵深防御的错觉 (The Illusion of Defence in Depth)

1.1 初衷与概念 (The Concept)

  • Idea: Multiple defences add (or multiply) together to improve security. If one defence is breached, there are others still standing.
    • 中文解析：纵深防御的初衷是多层防御叠加，假设一层被突破了，还有下一层可以阻挡攻击。就像中世纪的城堡（Castles），通常建有护城河、外城墙、内城墙和主塔等多重防御。

1.2 独立性陷阱 (The Independence Trap - 必考)

A critical assumption in Defence in Depth is that layers are independent (相互独立).

  • 盲目假设：人们常常盲目假设各个防御层是互相独立的，但现实往往并非如此。
  • 锁的隐喻 (The Locks Metaphor)：
    • One strong lock > Ten weak locks (1 把强锁 > 10 把弱锁)。
    • An attacker who can break a weak lock can break ten. (能破解一把弱锁的攻击者，同样能用同样的方法破解其余九把。)
    • A door with ten locks is weakened by ten sets of holes drilled in it. (而且，门上多打 10 个孔，反而增加了脆弱的攻击面 / Attack Surfaces，削弱了门本身的物理强度。)

2. 系统性失效案例 (Systemic Failure)

2.1 防火墙案例 (Firewalls and Attack Surfaces)

  • Scenario: You deploy a standalone (“hardware”) firewall in front of a computer, and also configure the operating system firewall securely on the computer. (假设你部署了独立硬件防火墙，又开启了服务器系统自带的防火墙。)
  • The Failure: If there is an authentication server which permits users to log in to both the firewall and the computer, an attacker who attacks the authentication server becomes admin on both.
    • 中文解析：如果这两个防火墙的登录鉴权依赖于同一个身份验证服务器（比如公司的 Active Directory），攻击者只需攻破该验证服务器，就能瞬间获得两层防火墙的 root 权限，轻松绕过所有规则。
  • 结论 (Conclusion)：共享单点故障 (Single Point of Failure) 会让所谓的纵深防御彻底崩塌。系统越复杂，涉及的设备越多，越难保证真正的独立性。

2.2 飞机引擎案例 (The Aircraft Engine Analogy)

Professor uses aircraft safety to illustrate that more redundant layers don’t always equate to safety.

  • Independence Failure: If a plane runs out of fuel, or enters a cloud of volcanic dust, all engines fail, whether there are one, two, three, four or eight engines.
    • 中文解析：如果飞机没油了，或者飞进了火山灰云层，无论你有几个引擎（几层冗余），它们都会同时失效。这就叫缺乏独立性的系统性失效。环境因素可以直接绕过你精心设计的“冗余层”。

3. 攻击树 (Attack Trees)

A method to model and visualize potential attack paths.

3.1 结构 (Structure)

  • Top (树顶): The attacker’s goal. (攻击者的最终目标，如：窃取客户数据 / Steal customer data)。
  • Branches (向下延伸): The various ways they might achieve that descending from it. (各种可能的手段分支，如：偷备份 / Obtain backup media, 拦截邮件 / Intercept email, 黑客渗透 / Hack fileserver, 贿赂管理员 / Bribe admin)。
  • Nodes: Each leaf can have an “Attack cost”. (每个叶子节点可以附带一个攻击成本或概率指标)。

3.2 难点与盲区 (Challenges & Blind Spots)

  • 难点 (Difficulty)：
    • Building them is hard, looking for independence is hard too.
    • 中文解析：不仅构建攻击树本身很困难，在复杂的攻击树中寻找真正“独立”的防御节点也极度困难。
  • 盲区 (Blind Spots)：
    • Attack trees often miss attacks on “subsidiary protocols” (附属协议).
    • Classic example: DNS.
    • 中文解析：攻击者往往不按套路出牌，他们可能不会直接攻击你设防严密的数据库或主业务逻辑，而是专挑底层的“附属协议”（如篡改 DNS 记录进行劫持）下手，这往往是防御规划的盲区。

4. 📝 Exercise: Changing Marks (课堂练习：修改成绩)

Goal: Take five minutes and build an attack tree for changing your end of year marks on a marks database.
Assumptions:

  • Database is on a computer, behind a firewall.
  • Administered by an administrator.
  • Used by lecturer.

思考方向：不要局限于技术手段。可以通过黑客技术攻击防火墙，也可以直接贿赂/社工管理员，或者窃取讲师的凭证。这展示了攻击路径的多样性和攻击树分支的广度。

5. Conclusions: Complexity is Hard (结论：复杂性是分析的阻碍)

  • In general, the more devices and elements are involved in a security solution, the less likely that it is accurately analysed.
    • 中文解析：纵深防御通常意味着增加更多的设备和组件。但系统越复杂，越难以被准确分析。在“保持简单（易于分析）”和“纵深防御”之间并没有一个通用的标准答案，这是安全设计中需要不断权衡的难点。
